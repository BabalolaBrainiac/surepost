# No Emoji Policy

## Rule: ZERO EMOJIS IN AGENT CODE, CONFIGS, OR OUTPUTS

All agents in the surepost system must follow the wardscribe.io no-emoji policy.

### What to use instead:
- [done] - task complete
- [warning] - potential issue
- [failed] - error or failure
- [deployed] - deployment success
- [hot] - high priority item
- [secure] - security verified
- [blocked] - blocked or waiting
- [pending] - waiting to be processed

### Applies to:
- Agent configuration files (.md)
- Skill definition files (.yaml)
- API responses and error messages
- Log output
- Comments in code
- System prompts

### Exceptions:
- User-generated post content (the AI may generate posts WITH emojis if the user's style profile shows emoji usage)
- The content-generator skill respects user emoji preferences
